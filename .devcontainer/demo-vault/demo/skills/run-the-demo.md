# Skill — Take the demo tour

> A **skill** is a repeatable procedure an agent writes down the second
> time it does something, so it never re-derives the steps. This one is a
> friendly example.

## Trigger phrase

- "show me around gosidian"
- "what can this demo do"

## Steps

1. Read **[[demo/hot]]** to see what's in focus right now.
2. Open the **graph view** and find the hub notes (most links in/out).
   [[demo/README]] and [[demo/memory/architecture]] should stand out.
3. Run a **search** for `MCP` — jump to [[demo/memory/glossary#MCP]].
4. Open the **backlinks** panel on [[demo/memory/glossary]] and walk back
   to where each term is used.
5. Look at a **media note** ([[demo/media/architecture-diagram]]) and an
   **HTML note** ([[demo/interactive/lifecycle]]).

## Gotcha

The graph only looks sparse until you add links. Every `[[wikilink]]` you
write becomes an edge — see [[demo/memory/architecture#The link graph]].
